# Спецификация: определение обучающего видео

Дата: 2026-05-28  
Проект: Green A

## 1. Цель

Устойчиво определять, является ли ролик обучающим, чтобы:
- оценивать рейтинг только для учебного контента;
- помечать неучебный как `Not Available`;
- избегать ложных отказов, когда данных мало (пустые/короткие субтитры, краткое описание).

## 2. Источники сигналов (по убыванию ценности)

1. Транскрипт/автосубтитры.
2. Описание ролика.
3. Название ролика.
4. Сегментация (chapters/фрагменты) и визуальные наблюдения.

## 3. Классы

1. `educational`: ролик явно обучающий, рейтинг рассчитывается.
2. `uncertain`: похоже на учебный, но данных недостаточно/неполная учебная механика.
3. `non-educational`: неучебный или медийно-продажный формат.

Для UI:
- `educational` -> обычный рейтинг.
- `uncertain` -> рейтинг возможен, но с пониженной уверенностью.
- `non-educational` -> `Not Available`.

## 4. Education Score (0-100)

Классификация должна опираться не на один жесткий признак, а на итоговый `Education Score`. Score нужен, чтобы ранжировать ролики по учебной полезности и снижать количество ложных отказов для настоящих уроков.

### 4.1. Цель видео (0-25)

Главный вопрос:

После просмотра зритель узнает что-то новое и сможет это применить?

Ориентиры:
- лекция по физике: 25;
- урок программирования: 25;
- вебинар по продажам: 20-25;
- интервью с предпринимателем: 10-15;
- новости: 0-5;
- пранк: 0.

### 4.2. Наличие структуры обучения (0-15)

Высокий балл, если есть:
- введение;
- объяснение темы;
- примеры;
- выводы.

Сильные маркеры:
- "Сегодня разберем...";
- "Шаг 1";
- "Рассмотрим пример";
- "Подведем итог".

Лекция университета с понятной последовательностью может получить 15/15.

### 4.3. Плотность новых знаний (0-20)

Один из самых сильных критериев. Оценивается количество:
- определений;
- формул;
- фактов;
- терминов;
- концепций.

Ориентиры:
- "кот смешно упал": 0;
- интервью актера: 3-5;
- лекция MIT: 20.

### 4.4. Практическая применимость (0-10)

Высокий балл, если после просмотра зритель может что-то сделать:
- настроить сервер;
- паять;
- готовить;
- считать налоги;
- повторить упражнение или применить метод.

### 4.5. Доля объяснений в речи (0-10)

Сильный сигнал обучения, если в транскрипте много объясняющих конструкций:
- "потому что";
- "поэтому";
- "например";
- "означает";
- "рассмотрим";
- "заключается в том".

### 4.6. Наличие визуального обучения (0-10)

Высокий балл, если на экране есть:
- схемы;
- графики;
- формулы;
- презентации;
- демонстрации;
- экран компьютера с объяснениями.

### 4.7. Намерение автора (0-10)

Оценивается по названию и описанию.

Сильные маркеры:
- `Tutorial`;
- `Course`;
- `Webinar`;
- `Workshop`;
- `Training`;
- `Lecture`;
- `How to`;
- `Guide`;
- `Explained`;
- `урок`;
- `курс`;
- `лекция`;
- `мастер-класс`;
- `разбор`;
- `как`.

## 5. Уровни Education Score

### 5.1. 0-30: необучающее

Примеры:
- мемы;
- новости;
- музыка;
- фильмы;
- пранки;
- развлекательные влоги.

Поведение UI: итоговый рейтинг Green A не рассчитывается и показывается как `Not Available`. Внутренний `Education Score` можно показывать в диагностике.

### 5.2. 31-60: частично обучающее

Примеры:
- интервью;
- подкасты;
- документалки;
- разборы кейсов;
- повествовательные ролики с отдельными полезными объяснениями.

Поведение UI: рейтинг возможен, но с пониженной уверенностью и явным объяснением, какие учебные признаки найдены.

### 5.3. 61-100: обучающее

Примеры:
- курсы;
- лекции;
- вебинары;
- туториалы;
- мастер-классы;
- практические разборы с шагами и примерами.

Поведение UI: обычный рейтинг Green A.

## 6. Что считать не обучающим

Эти форматы должны автоматически снижать `Education Score`, если в них нет явной учебной механики:
- развлекательное: пранки, челленджи, реакции, влоги, стендап;
- художественное: фильмы, сериалы, клипы;
- новостное: `breaking news`, новости дня, оперативные сводки;
- игровое: `let's play`, стрим без объяснений, прохождение без обучения.

Важно: формат сам по себе не всегда запрещает рейтинг. Например, интервью или подкаст могут попасть в 31-60, если там есть объяснения, факты, кейсы и применимые выводы. Но развлекательный формат без объяснений должен оставаться в 0-30.

## 7. AI-анализ и входные данные

Для AI-анализа использовать минимум:
- название;
- описание;
- субтитры/транскрипт;
- первые 5 минут транскрипта;
- визуальные/OCR признаки, если доступны.

Ожидаемая точность при качественных субтитрах и описании: примерно 85-95%. Это целевой ориентир, а не гарантия для каждого ролика.

Модель должна учитывать минимум 10-15 признаков, а не один бинарный флаг. Итогом должен быть `Education Score`, уровень (`non-educational`, `uncertain`, `educational`) и короткое объяснение по основным факторам.

## 8. Обязательная учебная механика

Для уверенного `educational` нужны минимум 2 из 4 пунктов:
1. Метод/объяснение: разбор, шаги, демонстрация, алгоритм, why/how.
2. Практика/задания: exercise, quiz, задание, проверка.
3. Цель обучения: "вы научитесь", learning objective, результат урока.
4. Структура: главы/сегменты, последовательный учебный поток.

Это правило не должно перебивать высокий `Education Score`: если ролик набирает 61+ за счет цели, структуры, плотности знаний и объяснений, он считается обучающим даже без явных упражнений.

## 9. Позитивные сигналы

Усиливают вероятность `educational`:
- маркеры в названии: `урок`, `tutorial`, `lesson`, `гайд`, `how to`, `с нуля`, `для начинающих`, `part 1`;
- предметные маркеры: язык, программирование, математика и т.д.;
- признаки методики в описании/субтитрах: цель, пример, пошаговость, практика, проверка.

## 10. Негативные сигналы

Усиливают вероятность `non-educational`:
- медийный формат: интервью/подкаст/ток-шоу/реакция/влог;
- entertainment-маркеры: клип, концерт, юмор, стрим;
- агрессивный маркетинг: обещания "быстро/легко", сильный sales push без методики.

Негативные сигналы должны снижать score, но не должны автоматически обнулять ролик, если в транскрипте или описании есть сильные учебные признаки.

## 11. Режим "мало данных" (sparse evidence)

Если субтитры и описание короткие/почти пустые:
- не отправлять ролик автоматически в `non-educational`;
- при сильном учебном названии и отсутствии жестких негативов -> `uncertain`;
- переводить в `non-educational` только при явных hard-negative признаках.

Это защищает учебные ролики от ложного отказа, когда YouTube не отдал полноценную транскрипцию.

## 12. Hard-negative правила

`non-educational` сразу, если:
1. Явно медийное название (`official music video`, `гость программы`) и нет учебной механики.
2. Сильные entertainment/sales признаки и нет цели/метода/практики.
3. Мотивационно-продажный формат с быстрыми обещаниями без учебного ядра.

Hard-negative правило должно применяться только после проверки title/description/transcript на учебную цель, структуру, объяснения и плотность знаний.

## 13. Примеры для валидации

### Educational
1. `Урок SQL: JOIN на практике` + цель + упражнение + проверка.
2. `Грузинский язык с нуля — урок 1` + словарь + упражнение в описании.
3. `Linear Algebra Lecture` + examples + exercises.

### Uncertain
1. `Грузинский язык с нуля — урок 1`, но субтитры и описание почти пустые.
2. `Разбор темы: event loop`, есть объяснение, но без практики/проверки.

### Non-educational
1. `Интервью с экспертом`, разговор без учебной структуры.
2. `Official music video`.
3. `Стань экспертом за 7 дней` + сильный sales push без методики.

## 14. Критерий приемки

1. Не менее 90% учебных роликов из тестового набора не попадают в `non-educational`.
2. Не менее 90% явных интервью/влогов/клипов попадают в `non-educational`.
3. Для sparse-кейсов доля `uncertain` выше доли ложного `non-educational`.
4. Каждый анализ возвращает `Education Score` 0-100 и объяснение по основным критериям.
5. Явные уроки, лекции, вебинары и туториалы не должны получать ниже 61 без сильных hard-negative признаков.
6. Очевидные развлекательные ролики без объяснений должны попадать в диапазон 0-30.
7. Частично полезные интервью, подкасты и документалки должны чаще попадать в диапазон 31-60, а не автоматически в `non-educational`.
