# Спецификация: определение обучающего видео

Дата: 2026-05-28  
Проект: Green A

## 1. Цель

Устойчиво определять, является ли ролик обучающим, чтобы:
- оценивать рейтинг только для учебного контента;
- помечать неучебный как `Not Available`;
- избегать ложных отказов, когда данных мало (пустые/короткие субтитры, краткое описание).

## 2. Источники сигналов (по убыванию ценности)

1. Транскрипт/автосубтитры.
2. Описание ролика.
3. Название ролика.
4. Сегментация (chapters/фрагменты) и визуальные наблюдения.

## 3. Классы

1. `educational`: ролик явно обучающий, рейтинг рассчитывается.
2. `uncertain`: похоже на учебный, но данных недостаточно/неполная учебная механика.
3. `non-educational`: неучебный или медийно-продажный формат.

Для UI:
- `educational` -> обычный рейтинг.
- `uncertain` -> рейтинг возможен, но с пониженной уверенностью.
- `non-educational` -> `Not Available`.

## 4. Обязательная учебная механика

Для уверенного `educational` нужны минимум 2 из 4 пунктов:
1. Метод/объяснение: разбор, шаги, демонстрация, алгоритм, why/how.
2. Практика/задания: exercise, quiz, задание, проверка.
3. Цель обучения: "вы научитесь", learning objective, результат урока.
4. Структура: главы/сегменты, последовательный учебный поток.

## 5. Позитивные сигналы

Усиливают вероятность `educational`:
- маркеры в названии: `урок`, `tutorial`, `lesson`, `гайд`, `how to`, `с нуля`, `для начинающих`, `part 1`;
- предметные маркеры: язык, программирование, математика и т.д.;
- признаки методики в описании/субтитрах: цель, пример, пошаговость, практика, проверка.

## 6. Негативные сигналы

Усиливают вероятность `non-educational`:
- медийный формат: интервью/подкаст/ток-шоу/реакция/влог;
- entertainment-маркеры: клип, концерт, юмор, стрим;
- агрессивный маркетинг: обещания "быстро/легко", сильный sales push без методики.

## 7. Режим "мало данных" (sparse evidence)

Если субтитры и описание короткие/почти пустые:
- не отправлять ролик автоматически в `non-educational`;
- при сильном учебном названии и отсутствии жестких негативов -> `uncertain`;
- переводить в `non-educational` только при явных hard-negative признаках.

Это защищает учебные ролики от ложного отказа, когда YouTube не отдал полноценную транскрипцию.

## 8. Hard-negative правила

`non-educational` сразу, если:
1. Явно медийное название (`official music video`, `гость программы`) и нет учебной механики.
2. Сильные entertainment/sales признаки и нет цели/метода/практики.
3. Мотивационно-продажный формат с быстрыми обещаниями без учебного ядра.

## 9. Примеры для валидации

### Educational
1. `Урок SQL: JOIN на практике` + цель + упражнение + проверка.
2. `Грузинский язык с нуля — урок 1` + словарь + упражнение в описании.
3. `Linear Algebra Lecture` + examples + exercises.

### Uncertain
1. `Грузинский язык с нуля — урок 1`, но субтитры и описание почти пустые.
2. `Разбор темы: event loop`, есть объяснение, но без практики/проверки.

### Non-educational
1. `Интервью с экспертом`, разговор без учебной структуры.
2. `Official music video`.
3. `Стань экспертом за 7 дней` + сильный sales push без методики.

## 10. Критерий приемки

1. Не менее 90% учебных роликов из тестового набора не попадают в `non-educational`.
2. Не менее 90% явных интервью/влогов/клипов попадают в `non-educational`.
3. Для sparse-кейсов доля `uncertain` выше доли ложного `non-educational`.
